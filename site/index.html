The real resume() renders the packet into the Markdown briefing a fresh agent is handed as its opening context — goal, background to respect, what's done, the live state and files to open first, next steps, open questions, artifacts, and a provenance footer. Empty sections render an italic placeholder so the resumer sees exactly what was not handed over.

        

      


      
      

        
The baton is a file. Agent A packs the packet and stops; agent B — a different model, harness, or machine — reads the resume() briefing and runs. Then B resolves a question, makes progress, and checkpoints through the real revise() (append + version-bump + re-hash) — and diffPackets(v1, v2) reports exactly what advanced.

Quickstart — adopt it in a minute

    
Three ways to pass the baton.

    
A CLI for one-off packing and resuming, a library for your own agent runtime, or an adapter that drafts a packet straight from a harness transcript.

    

      

        ① CLI
        
Pack & resume

        
Pack the current working state into a packet, then print the briefing a fresh agent should read.

        
# no install needed
npx @avee1234/handover pack \
  --goal "ship OAuth PKCE" \
  --out handoff.json

npx @avee1234/handover resume \
  handoff.json

      

      

        ② Library
        
In your runtime

        
Pack, score, brief and diff inside your own agent loop. Zero dependencies, pure ESM.

        
import { pack, resume,
  completeness } from
  '@avee1234/handover';

const p = await pack(state);
if (completeness(p).score < 1)
  warn('thin handoff');

      

      

        ③ Adapter
        
From a transcript

        
Draft a packet from a Claude Code session, refine it, then pack the real thing.

        
import { fromClaudeCode }
  from '@avee1234/handover';

const draft =
  fromClaudeCode(jsonl);
// refine → pack(draft)

Meta

    
Built — and handed off — by an autonomous agent factory.

    

      
handover was triaged, specced, implemented, adversarially reviewed, and released by an autonomous "software factory" of agents, with a human approving the gates. It exists because the factory itself needed it: when one agent's session ends and another picks up the work, the baton has to be a real, portable artifact — not a lossy summary. The tool that lets agents resume each other's work was built by agents resuming each other's work.
